A. DDL (Data Definition Language)

DDL, atau Data Definition Language, digunakan untuk mendefinisikan struktur dan skema database. Ini termasuk membuat, mengubah, dan menghapus objek-objek dalam basis data seperti tabel, indeks, dan tampilan.

Contoh perintah DDL yang umum digunakan:

  • CREATE: Digunakan untuk membuat objek baru dalam basis data. 
    CREATE TABLE Mahasiswa (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nama VARCHAR(100),
    umur INT
);
  • ALTER: Digunakan untuk mengubah struktur objek yang sudah ada dalam basis data. 
    ALTER TABLE Mahasiswa
ADD COLUMN alamat VARCHAR(255);
  • DROP: Digunakan untuk menghapus objek dari basis data. 
    DROP TABLE Mahasiswa;

B. DML (Data Manipulation Language)

DML, atau Data Manipulation Language, digunakan untuk memanipulasi data yang ada dalam basis data. Ini mencakup operasi seperti menambah, mengubah, menghapus, dan mengambil data dari tabel.

Contoh perintah DML yang umum digunakan:

  • INSERT: Digunakan untuk menambahkan data baru ke dalam tabel. 
    INSERT INTO Mahasiswa (nama, umur, alamat)
VALUES ('Budi', 20, 'Jl. Merdeka No. 123');
  • UPDATE: Digunakan untuk mengubah data yang sudah ada dalam tabel. 
    UPDATE Mahasiswa
SET umur = 21
WHERE nama = 'John';
  • DELETE: Digunakan untuk menghapus data dari tabel. 
    DELETE FROM Mahasiswa
WHERE umur > 25;
  • SELECT: Digunakan untuk mengambil data dari tabel. 
    SELECT * FROM Mahasiswa;

C. DCL (Data Control Language)

DCL, atau Data Control Language, digunakan untuk mengontrol hak akses dan izin pengguna terhadap objek database. DCL mencakup perintah-perintah seperti GRANT dan REVOKE.

Contoh perintah yang umum digunakan:

  • GRANT: Digunakan untuk memberikan izin kepada pengguna untuk melakukan operasi tertentu pada objek database. 
    GRANT SELECT, INSERT, UPDATE ON Mahasiswa TO 'user1'@'localhost';
  • REVOKE: Digunakan untuk mencabut izin yang telah diberikan sebelumnya kepada pengguna. 
    REVOKE INSERT ON Mahasiswa FROM 'user1'@'localhost';
